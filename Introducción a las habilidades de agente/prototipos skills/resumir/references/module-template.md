# Module Template

Use this template as the structural reference for every module you write.

## File title

```
## Módulo X: Título descriptivo
```

## Section structure

### [Descriptive title for the section]

Start with a short conceptual paragraph explaining the core idea in plain language.

Then include, as relevant:
- Code blocks with inline comments for practical examples
- Comparison tables for properties, parameters, or options (always include these — even if the source material doesn't have them)
- Diagrams as text/ASCII when useful
- Callout blocks (>) for rules of thumb, warnings, or key insights

### Repaso — Módulo X

5–6 questions that test conceptual understanding, not just recall.
Mix "what", "why", and "what would you do if" questions.

### Ejercicio — Módulo X

One practical exercise the reader can actually do.
Include a template or starter code when helpful.

## Heading uniqueness rule

All headings must be unique across the entire file.
If a heading would repeat (e.g. "Repaso"), always append the module number:
- "Repaso — Módulo 2" ✓
- "Repaso" ✗ (if it already exists in another module)
